# yarn-swap — Audit (2026-06)

Dimensions run: the three core dimensions (correctness; security & provenance via `pi_audit.py` + a manual git-provenance check; maintainability & dependencies), plus **JS conventions** and **accessibility** because this is a single-file, user-facing web UI. Skipped as not applicable: regulatory defensibility, statistical validity (the matcher is a disclosed heuristic, not an inferential estimator), financial-data correctness (the price field is a static 1–3 tier label, no money/FX), and citation & methods integrity (no bibliography or methods doc).

**Summary — healthy.** `index.html` is a clean, fully self-contained prototype with no external scripts, no network calls, and no browser-storage use; the 45-yarn dataset is internally consistent (every blend sums to 100%, every fiber maps to a known family, weight coverage is now reasonable). The scoring code is correct on the paths I traced — no division-by-zero, no score exceeds 100, the wool/merino pooling math is sound, and the thickness metric is now symmetric. `pi_audit` produced **4 findings, all false positives**, and git `fsck` is clean. Notably, every High/Med item from the prior `docs/AUDIT.md` (contrast failures, missing `aria-live`, missing input validation, the arbitrary fiber-remainder guess, missing `escapeHtml`, inline `onclick`) has since been **fixed in the code** — but that same `docs/AUDIT.md` was never updated, so it now contradicts the codebase. That stale audit document is the single most consequential thing to address.

## Major
- `docs/AUDIT.md` — the committed prior audit (dated 2026-06-05) is now **substantially stale and actively misleading**. It describes the 38-yarn version (the array now holds 45), its line references are all off by ~50–80 lines (e.g. it cites `getTarget()` at "line 320"; it now starts at `index.html:394`), and it asserts as *present-tense bugs* several things that have since been fixed: the arbitrary acrylic/wool fiber-remainder guess (`AUDIT.md:15-19` — code now uses the explicit `sFiber2` dropdown, `index.html:398-403`), missing input validation (`AUDIT.md:21` — now `validateSpecs()`, `index.html:366-392`), asymmetric thickness (`AUDIT.md:25` — now symmetric, `index.html:292-295`), missing `aria-live` (`AUDIT.md:66` — now present, `index.html:195`), and the latent-XSS/`escapeHtml` gap (`AUDIT.md:74` — helper now exists at `index.html:262-263`). A reader trusting this doc would "re-fix" already-fixed code or distrust the whole `docs/` trail. Fix: regenerate or date-stamp it as superseded (e.g. add a header "superseded by yarn-swap-audit-2026-06.md") and correct the 38→45 count, or delete it in favor of this report.

## Minor
- `docs/yarn-tracker-state.json` — task line references are **systematically stale** by the dataset-growth offset (e.g. task 2 cites `index.html:194-198` for `WEIGHTS`, actually `204-208`; task 3 cites `252-256` for `score()`, actually `311-330`; task 6 cites `266-276` for the fiber term, actually `299-309`). Notes are otherwise accurate. Fix: refresh the line citations, or drop exact line numbers in favor of function names that don't drift.
- `index.html:461` — displayed score is capped at 99 via `Math.min(s,99)` while the badge class and the 55-point threshold use the true score, so an exact specs-mode re-entry scores 100 internally but renders "99%". Cosmetic; carried over unfixed from the prior audit. Fix: allow 100, or document the intentional cap.
- `index.html:99` — buy-link **hover** state sets `--accent` (#b5485d) text on `--accent-soft` (#f6e3e7) background = **4.21:1**, which fails WCAG AA for normal-size text (`.buylinks a` is .85rem). This is the one residual contrast issue; the prior audit's other contrast failures (`--muted`, `--ok` badge, active tab) now all pass (5.0–5.9:1). Fix: darken the hover text or lighten the hover background.
- `index.html:281,321` — the `gaugeUnknown: 8` neutral credit (just over half of the 15-point gauge term) only fires for a custom target with no gauge, giving a gauge-less custom yarn a quiet edge over a real candidate that matched gauge poorly. Methodology note, not a bug. Fix: sanity-check the value against real substitutions during the planned scoring-tuning pass (tracker task 22).
- Scoring weights (`PTS`, `index.html:275-283`) and slopes are reasonable and now well-named, but remain **heuristic and unvalidated** against known-good real-world substitutions (tracker task 22, not started). Worth validating before any public launch, since the matcher's credibility rests on it.
- No automated tests. The scoring helpers (`score`, `fiberSimilarity`, `thicknessDiff`, `famPct`) are pure and trivially testable; a handful of assertions (self-score = 100, weight-gap-of-2 returns null, a known ordered substitution pair) would lock in behavior before the planned re-tuning. Fix: add a tiny test block or a `tests/` HTML harness.
- `.gitignore` ignores `node_modules/` and `*.log`, but the project has no Node toolchain or build step (it's a static file). Harmless, but slightly misleading about the stack. Fix: trim to what the project actually produces, or leave as a deliberate template.
- **Provenance:** author identity is inconsistent across history — `12baa04` is `shepherd70@gmail.com`, `8152edf` is `shepherd70@users.noreply.github.com` (no ID prefix), and `106b92f` is `12345678+shepherd70@users.noreply.github.com`, where `12345678` is a placeholder-shaped GitHub numeric ID that doesn't match the real one (`134653832+...`) on the merge commit `04d8ed2`. No sign of compromise — the `Co-Authored-By: Claude Fable 5` trailer and the gmail on the initial commit are consistent with legitimate AI-assisted authorship — but commit `106b92f` looks to have been made under a misconfigured git identity. Fix: set `user.email` to the real GitHub noreply address for future commits.
- No Content-Security-Policy. Fine for a local/static prototype, but the inline `<style>`/`<script>` will need a CSP (and would force `unsafe-inline` or external extraction) at deploy time (tracker task 32). Not actionable now.
